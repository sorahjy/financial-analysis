# 游资题材股形态识别设计稿

本文档记录后续“用 K 线和本地数据匹配游资埋伏、拉升、出货套路”的设计思路。它不是荐股规则，也不把任何个股走势归因为真实席位行为；这里的“游资/主力”只是一组可观察的价量行为假设。

核心目标：

- 在启动前识别 `ambush`：低位、筹码收敛、温和放量、题材未高潮。
- 在刚启动时识别 `ignite`：首阳/首板、箱体突破、板块共振。
- 在接力阶段识别 `relay`：分歧转一致、二波回踩再起。
- 在后段识别 `distribution` / `dump_risk`：高潮加速、天量滞涨、高位横盘派发、假突破。

## 1. 现有可用数据

### 1.1 股票日线与估值

来源：`data/stock_data.sqlite3`

表：`stock_history`

- `code`, `date`
- `daily_open`, `daily_high`, `daily_low`, `daily_close`
- `daily_volume`, `daily_amount`
- `daily_change_pct`, `daily_turnover_rate`
- `market_cap`, `pe_ttm`, `pe_static`, `pb`, `pcf`

可做的事：

- K 线结构：实体、上影线、下影线、振幅、跳空、突破、跌破、回收。
- 量能结构：成交额均线、量比、地量、天量、放量滞涨、缩量回踩。
- 换手结构：温和换手、换手脉冲、连续高换手、换手衰竭。
- 位置结构：近 60/120/250 日分位、距离新高/新低、回撤、涨幅。
- 市值约束：小市值题材池优先；大市值更容易变成趋势/机构逻辑。

### 1.2 行业与题材候选

来源：`data/stock_data.sqlite3`

表：`sw3_member`, `sw3_segment`

- `segment_code`
- `market_cap_yi`
- `official_market_cap_ratio`
- `is_leader`
- `sw3_segment.segment_name`
- `sw3_segment.parent_segment`

可做的事：

- 候选池：先用 `is_leader=1`，再按 `market_cap_yi <= 300` 筛“小盘题材龙头”。
- 静态行业：展示股票所属申万二级/三级，避免前端 unknown。
- 规模因子：有 `market_cap_yi` 用市值；同一分组市值缺失时可退回 `official_market_cap_ratio`。

### 1.3 SW2 行业日线

可做的事：

- 退潮判断：行业指数跌破短均线、题材换手高位回落、成交占比下降。

来源：`data/capital/`

- `theme_rankings.csv`：SW2 题材热度全量排名。
- `hot_money_ambush.json`：现有主力资金潜伏分输出。
- `hot_money_verify.json`：潜伏分后验验证结果。

可做的事：

- 形态匹配器不要从零选股，先复用 `theme_candidates` 做候选池。
- 后续 `stock_hot_money_radar.py ambush` 可复用这里的形态分作为子因子。
- `verify` 要用历史截面 PIT 重算，避免把未来行业热度或未来 K 线带进去。

## 2. 总体框架

我建议把短线题材股拆成四层，不直接预测“涨跌”：

1. `universe`：这只股票是不是游资可能感兴趣的池子。
2. `state`：现在处在吸筹、点火、接力、派发风险里的哪一段。
3. `trigger`：有没有从状态转入交易窗口的确认信号。
4. `risk`：这个信号是不是已经太晚、太拥挤、太容易被砸。

最终输出不要只有一个分数，建议形如：

```json
{
  "code": "600667",
  "name": "太极实业",
  "pattern_stage": "dump_risk",
  "ambush_score": 42.1,
  "ignite_score": 78.4,
  "relay_score": 61.5,
  "distribution_score": 82.3,
  "best_pattern": "高潮加速后天量风险",
  "evidence": ["20日涨幅过高", "换手分位98%", "接近60日新高"],
  "invalidations": ["跌回突破位", "题材指数退潮"]
}
```

## 3. 形态库

### 3.1 低位箱体吸筹

直觉：

- 股价长期下跌或长期横盘后，不再创新低。
- 区间振幅收窄，回撤变小。
- 成交额/换手率温和抬升，但价格没有明显上涨。
- 这是最接近“还没被市场看见”的埋伏段。

可用数据：

- `stock_history.daily_close`
- `daily_high`, `daily_low`
- `daily_amount`, `daily_turnover_rate`
- `market_cap`

识别条件：

- 近 120 日价格分位低于 45。
- 近 40 日高低点区间振幅低于近 120 日振幅的 60%。
- 近 20 日成交额均值大于近 60 日成交额均值，但不超过 2.2 倍。
- 近 20 日涨幅不超过 20%，避免已经启动。
- 近 20 日换手分位缓慢抬升，而不是单日爆量。

失败信号：

- 跌破箱体下沿后 3-5 日内无法收回。
- 放量突破后又收回箱体，并且成交额维持高位。
- 所属题材同步走弱。

### 3.2 低位筹码单峰密集

直觉：

- 通过换手率衰减重建成本分布。
- 如果筹码主峰在低位，且主峰附近筹码集中，说明近期筹码成本趋同。
- 当前价距离主峰不能太远，太远说明已经拉升，太低说明还没站回成本区。

已有代码：

- `stock_hot_money_radar.py` 里的 `_chip_metrics()`。
- 使用三角分布把每日换手铺到 `low - avg - high` 区间。

识别条件：

- `chip_concentration` 高。
- `chip_peak_pctile` 低。
- `chip_price_to_peak` 在合理区间，例如 -8% 到 +22%。
- `chip_winner` 中低位，不是大面积获利盘。

失败信号：

- 当前价显著低于主峰，说明套牢压力没有化解。
- 当前价显著高于主峰，且换手/成交额进入历史高分位，可能已经从吸筹转为派发。

### 3.3 假跌破洗盘后收回

直觉：

- 横盘末端故意跌破平台或前低，制造破位恐慌。
- 随后快速收回箱体，并且收盘靠近当日上半部。
- 这个结构比单纯突破更像“左侧洗盘完成”。

识别条件：

- 前面存在 30-80 日箱体。
- 某日最低价跌破箱体下沿 1%-5%。
- 1-5 个交易日内收盘重新回到箱体。
- 回收日 CMF 或收盘位置较强。
- 跌破时成交不应是极端天量；极端天量可能是真破位。

失败信号：

- 跌破后连续收在箱体下方。
- 回收后不能站稳中枢。
- 行业指数同步破位。

### 3.4 极致缩量压缩后放量突破

直觉：

- 游资喜欢“点火成本低”的票：波动压缩，筹码安静。
- 一旦突破，容易引发跟风盘。

识别条件：

- 近 20 日 ATR/振幅分位低于近 250 日的 25%。
- 近 20 日成交额分位低于 40%，但不能完全无量。
- 当日收盘突破近 40 日高点。
- 当日成交额大于近 20 日均值 1.5 倍以上。
- 突破前价格位置不能太高，近 120 日分位最好低于 70。

失败信号：

- 突破日长上影，收盘回到区间内。
- 次日低开低走，跌回突破位。
- 成交额暴增但涨幅很小，属于放量滞涨。

### 3.5 低位首阳 / 首板点火

直觉：

- 第一根强 K 往往是资金从潜伏转入明牌的信号。
- 最有价值的是“低位首阳”，不是已经连涨后的追高。

识别条件：

- 当日涨幅接近对应涨跌停幅度，或涨幅大于 7% 且收盘接近最高价。
- 过去 20 日没有类似大阳/涨停。
- 当日成交额大于 20 日均值 1.8 倍。
- 近 60 日涨幅不能过高。
- 所属 SW2 行业处于热度前列，或个股明显领先行业。

失败信号：

- 次日低开低走，无法站上首阳实体中位。
- 首阳当天是历史天量且上影较长。
- 板块热度没有跟随。

### 3.6 分歧转一致接力

直觉：

- 短线资金不是完全怕高，而是怕高位没有承接。
- 高换手日如果还能收强，说明分歧被市场消化。

识别条件：

- 前 3-8 日已有明显涨幅，但还没有进入极端高潮。
- 当日振幅较大、换手较高。
- 收盘位于当日高低区间上 70%。
- 成交额放大但没有超过近 250 日极端分位。
- 次日没有快速跌破分歧日低点。

失败信号：

- 高换手后不能新高。
- 高换手日长上影，次日低开。
- 所属题材指数同步缩量回落。

### 3.7 二波回踩再起

直觉：

- 第一波已经证明有资金关注。
- 真强票回调时缩量，不轻易跌破突破位或关键均线。
- 二次放量突破前高时，比第一次更容易被市场识别，但风险也更高。

识别条件：

- 第一波 10-30 日涨幅大于 25%。
- 随后回撤 20%-45%，成交额缩到第一波高点附近的 40%-70%。
- 回调不破第一波启动位或 20/30 日均线太久。
- 二次突破前高或右肩高点时放量。

失败信号：

- 回调不缩量。
- 二波不过前高，并且出现放量长上影。
- 第二波时行业热度已经退潮。

### 3.8 题材共振补涨

直觉：

- 游资主线常常先打辨识度最高的票，再扩散到低位补涨。
- 不能拿“已经上龙虎榜/已经大热”的票做埋伏，但可以用 SW2 热度识别资金正在流向哪里。

识别条件：

- SW2 行业 `theme_hot_score` 排名前列。
- 个股静态行业或交易相关行业与热题材匹配。
- 个股近 20 日涨幅显著低于行业，但近 5 日成交额/换手开始抬升。
- 个股市值在 30-300 亿之间更优。
- 个股还没有进入连续大涨或高潮换手。

失败信号：

- 题材指数已经连续大涨并高位放量回落。
- 个股只是跟随尾盘脉冲，没有持续成交。
- 补涨启动时已经出现历史天量。

### 3.9 高潮加速

直觉：

- 最容易让人误判成“强”的地方，其实常常是出货前后。
- 连续大涨、换手极高、成交额历史高分位、收盘接近新高，是高风险组合。

识别条件：

- 近 20 日涨幅大于 40%-60%。
- 最新换手率处于 250 日 90% 分位以上。
- 最新成交额处于 250 日 90% 分位以上。
- 股价距离 60 日高点小于 5%。
- K 线出现长上影、巨量阴线、炸板近似结构时风险加倍。

失败信号：

- 对“继续上涨”来说，次日不能快速修复高位分歧。
- 对“埋伏”来说，出现此形态直接排除。

### 3.10 高位横盘派发

直觉：

- 高位不再新高，但成交额维持很高。
- 每次冲高都有卖压，上影线变多。
- 这是游资出货更常见的慢形态。

识别条件：

- 过去 60 日涨幅较大。
- 近 20 日高点不再上移，或新高幅度很小。
- 近 20 日成交额均值仍处历史高分位。
- OBV/ADL 或 CMF 走弱。
- 上影线比例上升。

失败信号：

- 对空头判断来说，如果缩量回踩后重新放量突破前高，派发判断失效。
- 对多头交易来说，跌破高位箱体下沿且放量，风险确认。

### 3.11 拉高出货 / 假突破

直觉：

- 市场最容易被骗的是“放量突破”。
- 如果突破发生在高位、伴随历史天量、上影线和次日弱承接，往往不是启动，而是派发。

识别条件：

- 突破日成交额为近 250 日 95% 分位以上。
- 突破日上影线较长，收盘没有站稳突破位。
- 次日低开或跌回突破位。
- 行业热度已经排名高位但开始下降。
- 个股近 20/60 日涨幅已经显著跑赢行业。

失败信号：

- 次日强势反包并缩量站稳。
- 板块继续放量走强。

## 4. 建议的打分结构

### 4.1 候选池分

`universe_score = 0-100`

建议因子：

- 市值适配：30-300 亿最高。
- 数据完整度：近 250 日日线、换手率、成交额完整。
- 题材适配：属于 SW2 热门行业，或与热门行业收益/换手相关。
- 排除项：ST、极端低流动性、连续一字板、长期停牌。

### 4.2 埋伏分

`ambush_score = 0-100`

建议权重：

- 位置低位：20
- 箱体/波动压缩：15
- 温和放量但价格横住：20
- CMF/收盘买压：15
- 低位筹码单峰密集：25
- 题材早期共振：5

扣分：

- 20 日涨幅过大。
- 换手/成交额进入极端高分位。
- 长上影、放量滞涨。
- 所属题材已经高潮。

### 4.3 点火分

`ignite_score = 0-100`

建议权重：

- 首阳/首板强度：25
- 突破箱体有效性：25
- 放量质量：15
- 题材热度与共振：20
- 低位而非高位：15

扣分：

- 突破日历史天量。
- 上影线过长。
- 位置太高。
- 次日承接差。

### 4.4 接力分

`relay_score = 0-100`

建议权重：

- 分歧日承接：25
- 回踩缩量质量：20
- 二波结构：20
- 题材仍在主升：20
- 风险未高潮：15

扣分：

- 高位爆量不新高。
- 连续高换手。
- 题材成交占比下降。

### 4.5 派发风险分

`distribution_score = 0-100`

建议权重：

- 高位涨幅：20
- 历史天量/天换手：25
- 长上影/炸板近似：20
- 放量滞涨：15
- 题材退潮：20

这个分数不是买入分，而是风险分。`distribution_score >= 70` 时，无论其他分数多高，都应该标记 `dump_risk`。

## 5. 形态状态机

建议不要让每种形态互相独立输出，而是用状态机约束：

```text
base
  -> ambush       低位箱体/筹码集中/温和放量
  -> ignite       首阳/突破/首板
  -> relay        分歧转一致/二波
  -> distribution 高位横盘/放量滞涨
  -> dump_risk    高潮天量/假突破/题材退潮
```

状态转移示例：

- `ambush -> ignite`：突破箱体、首阳、成交额放大、题材不退潮。
- `ignite -> relay`：启动后缩量回踩不破，或高换手分歧日收强。
- `relay -> distribution`：高位放量但不再新高。
- `distribution -> dump_risk`：跌破高位平台、题材退潮、放量阴线。
- 任意状态 -> `invalid`：数据不足、停牌、极端流动性不足、题材映射失败。

## 7. 验证方式

验证不能只看 Top 股票未来涨没涨，要拆成几组：

### 7.1 横截面 Rank IC

每个 as-of 日：

- 用当日以前数据算分。
- 对同一日所有候选股票按分数排序。
- 计算未来 5/10/20/30/60 日收益的 Spearman Rank IC。

目的：

- 排除市场 beta。
- 看分数是否真的能在同一天的股票之间排序。

### 7.2 分位收益

每个 as-of 日把股票分成五组：

- Q1 低分。
- Q5 高分。

观察：

- Q5 是否持续高于 Q1。
- 多空差是否稳定。
- 命中率是否超过 50%。

### 7.3 风险验证

对 `distribution_score`：

- 看未来最大回撤。
- 看未来 5/10 日下跌概率。
- 看跌破 10/20 日线概率。

这个分数的目标不是涨幅，而是避险。

### 7.4 触发验证

对 `ignite` 和 `relay`：

- 不能只看触发日后收益。
- 要加入次日承接条件。
- 比较“触发但次日弱”和“触发且次日强”的差异。

## 8. 第一版落地优先级

优先级 1：先做 `ambush + distribution`

原因：

- 这两个最适合日线数据。
- 一个负责找早期，一个负责排除晚期。
- 不依赖实时盘口和 L2。

优先级 2：加入 `theme_candidates`

原因：

- 单只股票的 K 线容易产生大量假信号。
- 题材热度和个股相对行业强弱能过滤“无题材的技术形态”。

优先级 3：做 `ignite`

原因：

- 日线能判断首阳/突破，但不能看到盘中封板质量。
- 需要次日承接做二次确认，否则容易追到假突破。

优先级 4：做 `relay`

原因：

- 接力更依赖盘中分时和情绪周期。
- 只有日线时可以先做粗略版本，但不要过度相信。

## 9. 当前最重要的判断

我认为后续真正有价值的不是“找最强股票”，而是找：

1. 题材正在升温；
2. 个股还没高潮；
3. 市值和流动性适合短线资金；
4. 低位筹码已经收敛；
5. 有轻微资金痕迹但还没有被大众看到；
6. 一旦出现点火，能用次日承接确认，而不是盲目追突破。

换句话说，第一版目标应该是：

```text
小盘题材池
  + 低位筹码收敛
  + 温和放量不涨
  + 板块早期共振
  - 高潮换手
  - 天量长上影
  - 高位派发
= 游资潜伏观察清单
```

后续代码只要能稳定输出这张清单，并用历史 PIT 验证它不是未来函数，就已经比单纯看涨停、龙虎榜、人气榜更接近你的经验判断。
